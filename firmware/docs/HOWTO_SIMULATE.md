# HOWTO_SIMULATE

## Есть ли эмулятор для этого проекта?
Да — самый практичный вариант для ESP32-S3 это **Wokwi**.

Важно: эмуляция полезна в первую очередь для проверки:
- базовой загрузки прошивки,
- логики FSM,
- serial-логов,
- общего поведения loop/tick.

Для точной валидации железа (реальные I2C-сенсоры, тайминги, шум кнопок, питание, SPI-дисплей) нужен физический стенд.

## Вариант 1: быстрый тест без железа (unit tests)
Из папки `firmware`:

```bash
cd firmware
pio test
```

Это проверит platform-independent часть (domain/FSM), которая не зависит от реальных датчиков/дисплея.

## Вариант 2: эмуляция в Wokwi

### 1) Установить Wokwi CLI
Официальный инструмент: `wokwi-cli`.

### 2) Подготовить бинарник через PlatformIO
```bash
cd firmware
pio run
```

### 3) Создать конфиг Wokwi
Нужны как минимум:
- `diagram.json` (виртуальная схема),
- `wokwi.toml` (путь к firmware ELF/BIN).

### 4) Запустить эмуляцию
```bash
wokwi-cli --interactive
```

## Ограничения эмуляции
- Поведение ADXL345/BH1750 может отличаться от реального устройства.
- ST7789 может требовать упрощенной конфигурации в симуляции.
- Debounce/hold/combo корректнее проверять на реальных кнопках.

## Рекомендованный workflow
1. Локально гонять `pio test` на каждое изменение domain/FSM.
2. Проверять базовый boot/serial/FSM в Wokwi.
3. Финальную проверку делать на реальном ESP32-S3 + сенсоры + дисплей.
